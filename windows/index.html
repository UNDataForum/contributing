Developing on Windows

Developing on Windows 10 is fully supported with Windows Subsystem for Linux (WSL). As a prerequisite, you need to ensure that you have local admin privileges.

Windows Subsystem for Linux

The Windows Subsystem for Linux (WSL) is Microsoft's attempt to relieve the pains that developers working with a non-Microsoft on Windows had for many nears. WSL basically gives a Linux development experience to Windows users.

Your starting point is the WSL Windows 10 Installation Guide in which you find instructions how to check if your Windows 10 is compatible. If you have access to the Microsoft Store, you can install Ubuntu 18.04 straight from the store.

If you do not have access to the Microsoft Store, you need to follow Manually download WSL distro packages to download Ubuntu 18.04. When the download is complete, you need to continue with Windows Server Installation Guide. Finally, you need to setup WSL as explained in Initializing a newly installed distro.

If at any point you have to roll back your WSL installation, it is not enough to delete the folder that contains the installation. Check for installed WSL distributions by running

wslconfig.exe /list

Pick your distirubtion, e.g. Ubuntu-18.04. Run

wslconfig.exe /u Ubuntu-18.04

to unregister. Now you can delete the folder containing the WSL distribution and start over.

At this point you have a Ubuntu 18.04 installation that you can access via the WSL terminal, but which is otherwise isolated from Windows 10.

Tooling

Now you can follow the steps in Developing on Linux to install all required tooling in your WSL Ubuntu 18.04.

VSCode

All tooling runs in WSL, except your editor, which runs on Windows. Therefore, go ahead and install the windows version for VSCode on your Windows 10.

Next download wslgit.exe and place it in your dev folder. Before configuring VSCode, we need to integrate WSL with VSCode by adding the following Windows-specific settings:

SettingValueComment
files.eol Change line endings to LF for WSL compatibility.
git.pathC:\\path\\to\\wslgit.exePath to WSLGit.
terminal.integrated.shell.windowsC:\\WINDOWS\\System32\\bash.exeUse Ubuntu bash via WSL as terminal.

You can now run Git in VSCode's terminal including authentication to GitHub as described in the Developing on Linux guide.

Choosing the right working directory

Your Windows home directory is %USERPROFILE%, which normally corresponds to C:\Users\<username>. In Linux your home directory is /home/<username>. Your WSL home directory and Windows home directory are not the same. This is crucial when cloning a repository with Git. If you just clone into /home/<username>, VSCode cannot access your project. This why WSL mounts all Windows system drives. %USERPROFILE% corresponds to /mnt/c/Users/<username>. If you clone in this directory (or better a subdirectory), VSCode can access your project.
